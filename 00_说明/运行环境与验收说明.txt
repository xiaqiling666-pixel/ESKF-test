运行环境与验收说明

这份说明只解决一个问题：
别人拿到当前项目后，
怎样先把环境配好，
怎样做低副作用检查，
怎样区分“检查入口”和“会生成结果的运行入口”。

一、推荐运行环境

当前项目是 Python 离线研究原型。

建议环境：
- Python 3.10 或更高版本
- Windows PowerShell 或命令提示符
- 在项目根目录执行命令

当前本机已验证环境：
- Python 3.13.2
- numpy 2.2.4
- pandas 2.2.3
- matplotlib 3.10.1

这不是强制锁死版本，
只是当前可运行环境记录。
项目依赖仍以根目录 `requirements.txt` 为准。

二、安装依赖

在项目根目录运行：

`python -m pip install -r requirements.txt`

当前只保留最小依赖：
- numpy
- pandas
- matplotlib

当前代码没有实际依赖：
- scipy
- pyyaml

因此暂时不把这些库写进 requirements。
后续如果真正引入对应功能，
再同步更新依赖文件。

三、环境检查入口

如果只是确认 Python 和依赖是否可用，
优先运行：

`04_tools\检查运行环境.bat`

或在终端运行：

`python -c "import numpy, pandas, matplotlib; print('ok')"`

这个入口不会运行主流程，
也不会生成结果文件。

四、日常开发验收入口

日常开发优先运行：

`python -m unittest discover 05_tests`

或双击：

`04_tools\运行核心检查.bat`

这条路径只跑单元测试，
不应该生成新的结果目录或图表。

五、会生成结果文件的入口

下面这些入口会更新或生成结果文件，
不要在普通代码检查时频繁运行。

示例主流程：

`python 02_src\main.py`

或双击：

`04_tools\一键运行示例.bat`

真实样例配置：

`python 02_src\main.py 01_data\config_00000422_decoded.json`

或双击：

`04_tools\运行00000422_推荐配置.bat`

实验配置模板：

`python 02_src\run_experiment_batch.py`

或双击：

`04_tools\运行实验对比模板.bat`

这些入口对应的是运行结果验证，
不是日常最小检查。

六、当前验收标准

当前项目框架阶段，
最低验收标准是：

1. 环境检查能通过。
2. `python -m unittest discover 05_tests` 能通过。
3. 默认配置、样例配置和实验配置的角色不互相覆盖。
4. 运行入口和检查入口分清楚。
5. 不因为一次普通检查生成新的结果污染。

七、后续维护原则

如果新增依赖，
必须同步更新：
- `requirements.txt`
- 本说明文件

如果新增运行入口，
必须说明：
- 是否生成结果文件
- 输出目录在哪里
- 是否适合日常开发检查
